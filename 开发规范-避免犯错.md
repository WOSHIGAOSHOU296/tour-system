> 每犯一次错，记一条铁律。下次写代码前先读一遍。

## 一、Servlet/Filter 映射

### 🔴 铁律：只能用一个地方注册，绝对不能两头注册

| 方式 | 是否可用 |
|------|----------|
| `web.xml` 中 `<servlet>` + `<servlet-mapping>` | ✅ **只用这个** |
| Java 代码中 `@WebServlet("/xxx")` 注解 | ❌ **禁止使用** |
| Java 代码中 `@WebFilter("/xxx")` 注解 | ❌ **禁止使用** |

**犯错记录**：`UserController`、`ScenicController` 上写了 `@WebServlet` 注解，同时 `web.xml` 里也配置了映射，Tomcat 9 启动报错 "servlet不能映射为一个url模式"。

**正确做法**：
```java
// ✅ 正确 — 不加任何注解
public class XxxController extends HttpServlet { ... }
```
```xml
<!-- ✅ 正确 — 只在 web.xml 中配置映射 -->
<servlet>
    <servlet-name>XxxController</servlet-name>
    <servlet-class>com.tour.controller.XxxController</servlet-class>
</servlet>
<servlet-mapping>
    <servlet-name>XxxController</servlet-name>
    <url-pattern>/xxx</url-pattern>
</servlet-mapping>
```

## 二、数据库连接

### 🔴 铁律：JDBC URL 编码参数只能用 `UTF-8`

MySQL Connector 5.1.x 不支持 `utf8mb4` 这个编码名。

```java
// ❌ 错误
"jdbc:mysql://localhost:3306/tour_db?characterEncoding=utf8mb4&..."

// ✅ 正确
"jdbc:mysql://localhost:3306/tour_db?useUnicode=true&characterEncoding=UTF-8&useSSL=false"
```

## 三、导航栏链接

### 🔴 铁律：LoginFilter 必须同时配置精确路径和前缀路径

Servlet 映射 `/trip` 是精确路径，Filter 的 `/trip/*` 前缀匹配**不覆盖** `/trip` 这个裸路径。

```xml
<!-- ❌ 错误 — 只匹配 /trip/xxx，不匹配 /trip -->
<url-pattern>/trip/*</url-pattern>

<!-- ✅ 正确 — 两条都写 -->
<url-pattern>/trip</url-pattern>
<url-pattern>/trip/*</url-pattern>
```

**犯错记录**：`/trip?action=list` 未被 LoginFilter 拦截，未登录用户直接进入行程列表页面导致 NPE。

### 🔴 铁律：JSP 页面中跳转已实现模块时，必须指向 Servlet URL，不能用目录路径

```html
<!-- ❌ 错误 — 目录路径，没有 Servlet 处理 -->
<a href="strategy/">旅友圈</a>
<a href="trip/">行程规划</a>

<!-- ✅ 正确 — 指向 Servlet + action 参数 -->
<a href="strategy?action=list">旅友圈</a>
<a href="forum?action=list">旅途论坛</a>
<a href="scenic?action=list">景点浏览</a>
```

**检查清单**：每新增一个 Controller 后，必须回头检查 `common/header.jsp` 和所有 JSP 中指向该模块的超链接。

## 四、JSP 文件编辑

### 🔴 铁律：编辑 JSP 后检查三件事

| 检查项 | 说明 |
|--------|------|
| 变量不重复声明 | 同一个 `.jsp` 中每个变量名只能声明一次（`<%= xxx %>` 引用不需要声明） |
| import 不重复 | 检查文件头部 `<%@ page import=... %>` 不要有重复的类导入 |
| HTML 标签闭合 | 新增 `<div>`、`<form>` 等标签时确保闭合 |

**犯错记录**：往 `detail.jsp` 中添加留言板时，`List<Message> messages` 被声明了两次（第 11 行和第 12 行），导致编译错误。

## 六、新增模块标准流程

每次新增一个模块，按这个顺序来，避免遗漏：

```
1. DAO 层          → xxxDao.java         (CRUD 方法)
2. Service 层      → xxxService.java     (业务逻辑)
3. Controller 层   → xxxController.java  (请求分发，❌不加注解)
4. web.xml         → 添加 servlet/servlet-mapping
5. JSP 页面        → xxx/list.jsp 等
6. 检查 header.jsp → 导航栏链接是否指向正确的 Servlet URL
7. mvn compile     → 确认编译通过
8. 开发文档.md     → 更新状态、修改记录
```

## 七、代码编写规则

### 🔴 改 JSP 不改 Java，改 Java 检查 JSP

| 场景 | 影响范围 |
|------|----------|
| 改 CSS 样式 | 只影响视觉，不用动任何其他文件 |
| 改 JSP 的 HTML 结构 | 只改该文件，**保留所有 `<% %>` 数据逻辑不变** |
| 改 Controller 传给 request 的属性名 | **必须同步修改对应的 JSP 中被取出的属性名** |
| 改 Model 字段名 | **必须同步修改 DAO 中的 `mapXxx()` 方法和所有引用该字段的 JSP** |
| 新增 SQL 表 | 必须同步新增 Model 类 + DAO + 更新 init.sql |

### 🔴 Model 类命名

- 类名与数据库表行的 Java 命名对应，但不必完全一致
- 例如：`attractions` 表 → `Scenic` 类（文档叫景点）
- 字段名与数据库列名的映射在 DAO 的 `mapXxx()` 方法中处理
- 辅助字段（如 `cityName`、`username`）可以添加，用于联表查询结果

## 十二、CSS 优先级陷阱 — !important vs 内联样式

### 🔴 铁律：CSS 中 `!important` > HTML 内联 `style=""`

当 CSS 文件中写了：
```css
.nav-orange .navbar-nav > li > a {
    color: rgba(255,255,255,0.88) !important;
}
```
此时即使给 `<a>` 加上 `style="color: red;"` **也无效**，因为 `!important` 优先级高于内联样式。
按钮文字看不见的真正原因是：按钮背景是白色，但文字被上面这条规则强制刷成了半透明白色。

**正确做法**：为要特殊处理的元素写一条**更具体的**选择器 + `!important`：
```css
/* ✅ 选择器比 .navbar-nav > li > a 更具体，同等级别下更具体者胜出 */
.nav-orange .navbar-nav > li > a.nav-btn-fill {
    color: #e65100 !important;  /* 覆盖基类的 white */
}
```

**犯错记录**：改了三次（纯内联 style→加 CSS 类→改内联 style）都没生效，最后才发现是第33行基类 `!important` 在作祟。

## 十一、删除权限规则

### 🔴 铁律：用户可删除自己的内容，管理员可删除所有人的

所有删除操作必须同时检查"作者本人"和"管理员"两个条件：

```java
if (obj != null && (roleId == 4 || obj.getUserId().equals(user.getUserId()))) {
    // 允许删除
}
```

JSP 中按钮同理：
```jsp
<% if ((roleId == 4) || (sessionUser != null && obj.getUserId().equals(sessionUser.getUserId()))) { %>
    <a href="...">删除</a>
<% } %>
```

**适用**：论坛帖子、评论、景点留言、攻略。
**犯错记录**：之前所有删除逻辑只允许 roleId==4，普通用户无法删自己的内容。
