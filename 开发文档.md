## 四、架构说明

### 分层架构

```
浏览器 → Controller (Servlet) → Service → DAO → DB
              ↓
           JSP (View)
```

### 请求流转示例（登录）

```
POST /user?action=login
  → UserController.login()
    → UserService.login()          // 查用户、验密码、查角色
      → UserDao.findByUsername()   // SQL: SELECT * FROM users WHERE username=?
      → UserDao.findRoleIdByUserId()
    → session.setAttribute("user", user)
    → session.setAttribute("roleId", roleId)
    → resp.sendRedirect("index.jsp")
```

### 🔒 绝对不能动的核心约定

| 约定 | 原因 |
|------|------|
| **Session 属性名必须为 `"user"` 和 `"roleId"`** | 所有 JSP 和 Filter 都依赖这两个 key |
| **Controller 的 action 参数名不能改** | URL 中 `?action=xxx` 的所有 JSP 页面都硬编码了这个参数名 |
| **Model 类字段名必须与数据库列名对应** | DAO 中的 `mapUser(rs)` 等方法直接按列名取值 |
| **web.xml 中 Filter 的 url-pattern 不能随意删减** | 权限控制依赖 Filter 的拦截路径 |
| **header.jsp 和 footer.jsp 是公共模板** | 其他所有 JSP 通过 `<%@ include file="..." %>` 引入，路径不能动 |
| **DBUtil.java 中的数据库连接参数** | 改密码/地址只改 URL/USER/PASSWORD 三个常量，不要改动方法签名 |

### ✅ 可以改的部分

| 部分 | 说明 |
|------|------|
| CSS 样式 | `static/css/style.css` 随意调整 |
| 页面布局 | 各 JSP 中可以增删组件 |
| 前端 CDN 链接 | header.jsp 中的 Bootstrap/jQuery/ECharts 版本号 |
| SQL 种子数据 | `sql/init.sql` 中 INSERT 语句的内容 |
| pom.xml 依赖版本 | 升级 Servlet/MySQL 版本号（注意 API 兼容性） |
| 新增 Servlet/Filter | 在 web.xml 中添加新的映射 |
| **Servlet/Filter 映射方式** | ⚠️ 只能用 web.xml，绝对不能加 `@WebServlet` 或 `@WebFilter` 注解，否则 Tomcat 9 报重复映射错误 |
